**Consistency**\n   Use the same spacing, colors, typography everywhere\n\n### 2. **Hierarchy**\n   Make important elements larger and bolder\n\n### 3. **Simplicity**\n   Less is more; remove unnecessary elements\n\n### 4. **Professional**\n   Modern SaaS look; no amateur styling\n\n### 5. **Responsive**\n   Design works on all screen sizes\n\n### 6. **Accessible**\n   Ensure screen reader and keyboard support\n\n---\n\n## 💡 Pro Tips\n\n### Always Use Variables\n```css\n/* Bad */\npadding: 24px;\ncolor: #3b82f6;\n\n/* Good */\npadding: var(--spacing-lg);\ncolor: var(--primary);\n```\n\n### Follow BEM Naming\n```css\n/* Bad */\n.card-header-title\n.button-primary-large\n\n/* Good */\n.card__header-title\n.btn--primary.btn--lg\n```\n\n### Test Mobile First\n```css\n/* Structure for mobile first */\n.grid { grid-template-columns: 1fr; }\n\n/* Then enhance for desktop */\n@media (min-width: 768px) {\n  .grid { grid-template-columns: repeat(2, 1fr); }\n}\n```\n\n### Use Semantic HTML\n```html\n<!-- Bad -->\n<div class=\"header\">\n<div class=\"section\">\n\n<!-- Good -->\n<header class=\"layout__header\">\n<section class=\"page-section\">\n```\n\n---\n\n## 🚨 Common Mistakes to Avoid\n\n1. ❌ Hardcoding colors → ✅ Use var(--color)\n2. ❌ Inline styles → ✅ Use CSS classes\n3. ❌ Using emojis → ✅ Use Font Awesome\n4. ❌ Not responsive → ✅ Test mobile\n5. ❌ Inconsistent spacing → ✅ Use spacing variables\n6. ❌ Broken semantics → ✅ Use proper HTML tags\n7. ❌ Poor naming → ✅ Follow BEM convention\n8. ❌ Forgetting
